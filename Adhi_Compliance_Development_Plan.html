1. Project Overview & Objectives

Adhi Compliance is an AI-powered compliance management platform that enables organizations deploying AI systems to achieve and maintain regulatory compliance across multiple jurisdictions — EU AI Act, US state laws, India's DPDP Act, and China's AIGC provisions.

Primary Objectives

  1. Build a production-ready SaaS platform that auto-classifies AI systems by risk level and monitors compliance in real-time
  2. Launch an MVP within 6 months targeting Indian IT companies serving EU/US clients as the initial market
  3. Establish RootedAI as a first-mover in the India-focused AI compliance space
  4. Generate recurring revenue through tiered SaaS subscriptions (₹50K–₹2L+/month)
  5. Create defensible IP around regulation-as-data pipelines and compliance AI agents

Product Vision

Adhi Compliance is the compliance layer every AI company plugs into. One platform for risk classification, continuous monitoring, bias auditing, and regulatory reporting. Think of it as "Datadog for AI Compliance" — a horizontal platform that becomes essential infrastructure for any organization deploying AI.

Pivot Context: Adhi Compliance is being pivoted from a multimodal RAG system. The existing tech stack (FastAPI, Next.js, Celery/Redis, FAISS, Gemini) and features (multi-org support, file ingestion, RAG query engine) form the foundation. This plan covers the delta from current state to full compliance platform.

2. Scope Definition

In Scope (v1.0)

  • AI System Registry — register, catalog, and manage AI systems per organization
  • Risk Classification Engine — rule-based + LLM-powered EU AI Act risk classification
  • Regulation Knowledge Base — structured, searchable database of AI regulations (EU, US, India, China)
  • Compliance Chat (RAG) — natural language Q&A against regulation corpus
  • Compliance Dashboard — executive overview with scores, alerts, and status tracking
  • Bias Auditor — upload datasets, run fairness metrics, generate audit reports
  • Model Card Generator — auto-generate model cards from system metadata
  • Incident Reporting — EU AI Act Article 62 workflow with deadline tracking
  • Continuous Monitoring — scheduled compliance re-checks with alerting
  • Report Generation — PDF/Excel compliance reports and model cards
  • Multi-tenant SaaS — company isolation, user management, subscription tiers
  • Webhook Integrations — Slack, Teams, Email notifications

Out of Scope (v1.0)

  • Mobile native apps (PWA may be considered post-launch)
  • Real-time model monitoring (connecting to live ML inference pipelines)
  • Automated regulatory filing/submission to authorities
  • White-label / OEM distribution
  • Non-English regulatory document processing (v2.0 — Sarvam AI for Indian languages)
  • SOC 2 / ISO 27001 certification (post-launch priority)

4. Development Methodology

  • Framework: Agile Scrum (adapted for solo dev)
  • Sprint Length: 2 weeks
  • Ceremonies: Weekly self-review (Monday), sprint planning (start), sprint retro (end)
  • Tracking: GitHub Projects (Kanban board) with Issues linked to PRs
  • Branching: GitFlow — main (prod), develop (staging), feature/* branches
  • CI/CD: GitHub Actions — lint, test, build, deploy on merge
  • Documentation: Inline (JSDoc/docstrings) + Architecture Decision Records (ADRs) for major choices
  • Quality Gates: Every PR requires: passing tests, type-check clean, no lint errors

5. Phase 1: Planning & Foundation (Weeks 1–3)

Week 3: Foundation Code

  • Implement Supabase Auth integration (frontend + backend JWT validation)
  • Create SQLAlchemy models for all core tables
  • Run initial Alembic migrations
  • Implement RLS policies for multi-tenant isolation
  • Build API middleware stack: auth, rate limiting, CORS, error handling
  • Create shared UI component library: Button, Card, Input, Modal, Table, Badge
  • Build dashboard layout shell: sidebar, topbar, main content area

6. Phase 2: Core Platform Build (Weeks 4–10)

PHASE 2   Duration: 7 weeks (4 sprints)  ·  Focus: Core Features

Sprint 1: Auth + Company Profile + AI System Registry

Weeks 4–5  ·  2 weeks  ·  CORE

Backend Tasks:

  • Complete auth router: register, login, refresh, me, password reset
  • Company CRUD endpoints with validation
  • AI System CRUD endpoints with full metadata support
  • Pagination, filtering, sorting utilities
  • File upload endpoint for system documentation
  • Unit tests for all new endpoints (pytest)

Frontend Tasks:

  • Login/Register pages with Supabase Auth integration
  • Company Settings page with form validation
  • AI Systems list page (grid/table toggle, search, filters)
  • AI System detail page with tabs (Overview, Compliance, History)
  • New AI System registration wizard (multi-step form)
  • React Query hooks for all API calls

Acceptance Criteria: User can register → create company → add AI systems → view/edit/delete systems.

Sprint 2: Regulation Knowledge Base + Loader

Weeks 6–7  ·  2 weeks  ·  AI/ML

Backend Tasks:

  • Regulation model and CRUD endpoints
  • Regulation Loader Celery task: ingest PDF/HTML → parse → chunk → embed
  • Semantic chunking pipeline (512 tokens, 50 overlap, structural awareness)
  • Embedding generation with sentence-transformers (MiniLM-L6-v2)
  • pgvector storage and IVFFlat index configuration
  • Semantic search endpoint (vector similarity + metadata filtering)
  • Load initial regulation set: EU AI Act, NIST AI RMF, India DPDP, China AIGC

Frontend Tasks:

  • Regulation Library page with search, jurisdiction filters, status badges
  • Regulation detail page with article-level navigation
  • Compliance Chat page — streaming chat UI with source citations

Acceptance Criteria: 4+ regulations loaded and searchable. Chat returns accurate answers with article citations.

Sprint 3: Risk Classification Engine + Compliance Check

Weeks 8–9  ·  2 weeks  ·  AI/ML

Backend Tasks:

  • Rule-based pre-classifier (EU AI Act Annex III keyword matching)
  • LLM-based classifier (Gemini structured output with reasoning)
  • Classification confidence scoring and human review flagging
  • Compliance check engine: evaluate system against regulation requirements
  • Compliance score calculation algorithm
  • Multi-jurisdiction classification mapping
  • Celery tasks for async classification and compliance checks

Frontend Tasks:

  • Classification results display in AI System detail page
  • Risk level badges with color coding (red/orange/yellow/green)
  • Compliance Check results page with findings and recommendations
  • "Run Classification" and "Run Compliance Check" action buttons

Acceptance Criteria: System auto-classifies on registration. Compliance checks generate scored findings.

Sprint 4: Compliance Dashboard

Week 10  ·  1 week (sprint cleanup + dashboard)  ·  UI

Tasks:

  • Dashboard API: aggregated compliance metrics per company
  • Compliance Score Ring (animated donut chart — overall score)
  • Risk Distribution Heatmap (systems by risk level)
  • Recent Alerts Feed (real-time via Supabase Realtime)
  • System Status Grid (quick status cards per AI system)
  • Upcoming Deadlines widget
  • Sprint 1–3 bug fixes and polish
  • Internal demo / milestone checkpoint

Acceptance Criteria: Dashboard shows live compliance data. All Phase 2 features working end-to-end.

7. Phase 3: Advanced Features (Weeks 11–16)

PHASE 3   Duration: 6 weeks (3 sprints)  ·  Focus: Differentiation Features

Sprint 5: Bias Auditor + Model Card Generator

Weeks 11–12  ·  2 weeks  ·  AI/ML
  • Bias audit engine: demographic parity, equalized odds, disparate impact, calibration
  • Dataset upload and validation (CSV/JSON with protected attribute identification)
  • Statistical significance testing (Chi-squared, Fisher's exact)
  • LLM interpretation of bias metrics → actionable recommendations
  • Celery task for async audit execution
  • Model Card Generator: template-based + LLM-enhanced from system metadata
  • Bias Auditor UI: config form, progress indicator, results dashboard with charts
  • Model Card preview and PDF export

Sprint 6: Continuous Monitoring + Alerts

Weeks 13–14  ·  2 weeks  ·  INFRA
  • Celery Beat scheduler: hourly compliance drift detection
  • Regulation update checker (daily scrape of official sources)
  • Deadline tracker with countdown and grace period logic
  • Alert engine with priority levels (critical → SMS, high → email, medium → digest, low → in-app)
  • Webhook delivery system with HMAC signatures and retry logic
  • Notification preferences UI (per user: channels × event types)
  • Monitoring dashboard with real-time status indicators
  • Supabase Realtime integration for live alert feed

Sprint 7: Incident Reporting + Cross-Border Mapper

Weeks 15–16  ·  2 weeks  ·  CORE
  • Incident reporting workflow (Art. 62): create, triage, track, resolve
  • LLM-assisted severity assessment ("serious incident" classification)
  • Authority mapping (EU → AI Office, India → MEITY, etc.)
  • 15-day deadline tracker with automated reminders
  • Incident report auto-generation in required format
  • Cross-jurisdiction compliance mapper: unified view per AI system across all applicable regulations
  • Requirement gap analysis between jurisdictions
  • Incident UI: report form, timeline view, status tracker
  • Cross-border compliance matrix view

Sprint 8: Report Generator (PDF/Excel Export)

Week 16 (overlap)  ·  1 week  ·  CORE
  • Report templates: Compliance Summary, Full Audit Report, Model Card, Bias Report, Incident Report
  • PDF generation engine (WeasyPrint — HTML → PDF with branded templates)
  • Excel export for compliance matrices and raw audit data
  • Async report generation via Celery with download notification
  • Reports page UI with generation history and download links

9. Phase 5: Testing & QA (Weeks 23–25)

PHASE 5   Duration: 3 weeks  ·  Focus: Quality Assurance

Unit Testing

  • Backend: pytest with 80%+ coverage target on services and compliance engine
  • Frontend: Vitest + React Testing Library for component tests
  • API contract tests against OpenAPI spec

Integration Testing

  • End-to-end flows: registration → system creation → classification → compliance check → report
  • Celery task integration tests (regulation loading, bias audits, report generation)
  • Webhook delivery testing with mock endpoints
  • Multi-tenant isolation verification

Security Audit

  • OWASP Top 10 review: injection, XSS, CSRF, broken auth, misconfig
  • RLS policy audit: verify no cross-tenant data leakage
  • JWT handling review: token storage, refresh flow, expiry
  • Dependency vulnerability scan (Snyk/Dependabot)
  • API rate limiting and abuse testing

Performance Testing

  • Load testing: 100+ concurrent users, 1000+ API requests/minute
  • Vector search performance: query latency <200ms at 100K chunks
  • LLM response times: classification <10s, chat <5s first token
  • Report generation: PDF <30s, Excel <15s
  • Database query optimization (EXPLAIN ANALYZE on critical queries)

User Acceptance Testing

  • Recruit 5–10 beta testers from Indian IT companies
  • Structured feedback forms per feature area
  • Bug triage and priority fix cycle
  • UX iteration based on user feedback

16. Dependencies & Assumptions

Dependencies

  • Supabase — platform availability and pgvector support stability
  • Google Gemini API — continued availability and reasonable pricing for 1.5 Flash
  • Regulatory content — access to full text of EU AI Act, NIST AI RMF, India DPDP, China AIGC
  • AWS — ECS Fargate, ElastiCache, Secrets Manager availability in ap-south-1
  • Design contractor — availability during Phase 1 (Weeks 2–3)

Assumptions

  • Solo developer (founder) available full-time for the 28-week build period
  • AI coding assistants provide 1.5–2x productivity multiplier consistently
  • EU AI Act requirements remain stable through development (major provisions effective Aug 2025)
  • Indian IT companies serving EU/US clients represent a viable initial market
  • ₹50K–₹2L/month pricing is acceptable for the target market
  • Legal disclaimer ("not legal advice") is sufficient for v1.0; formal legal review before enterprise sales
  • Open-source bias/fairness libraries (AIF360, Fairlearn) provide sufficient algorithmic foundation
  • Infrastructure costs remain within $220–400/month range at launch scale
